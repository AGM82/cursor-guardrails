What cursor-guardrails is

A reusable project template that makes good engineering habits automatic. Written rules are advice; this template puts the real weight into checks a machine enforces and nobody can skip. github.com/AGM82/cursor-guardrails

The 7 guardrail tiers

TierMechanismLives inType
0 — WorkflowPlan Mode (Shift+Tab); plans saved to .cursor/plans/CursorDiscipline
1 — AdvisoryRules in .cursor/rules/*.mdc; cross-tool summary in AGENTS.mdRepoAdvisory
2 — ToolchainTypeScript strict, ESLint (security + a11y), Prettier, commitlint, testsRepoDeterministic
3 — RuntimeCursor hooks gate shell commands and secret reads; audit logRepoDeterministic
4 — AutomationHusky + lint-staged + gitleaks pre-commit; CI (typecheck, lint, test, build, audit, secret scan, SAST)Repo/CIDeterministic
5 — WorkflowsSlash commands: /review, /pr, /update-deps, /guardrail-upgradeRepoWorkflow
6 — ReviewAgent Review, Bugbot on PRs, TDD for logic-heavy workCursor/CISafety net

The 4 slash commands

CommandWhat it does
/reviewRun checks and review staged changes; report by severity before fixing anything
/prConfirm checks pass, write a Conventional Commit, push, and open a pull request
/update-depsUpdate dependencies one at a time, re-testing after each
/guardrail-upgradeLayers 0–7: bootstrap → baseline → gap analysis → upgrade → code audit → governance

Hard stops (always)

  • No secrets, credentials, or real client/personal data in source, logs, or commits
  • Validate input at every trust boundary; use parameterised queries only
  • Never log personal information
  • Do not modify tests to make failing code pass
  • Do not disable a linter or type rule to silence an error — fix the cause

Bootstrap prompt — paste into Cursor Agent chat

Copy this block and paste it into a new Cursor Agent chat in your existing project. It works day zero — nothing needs to be in place first.

Upgrade this project from the cursor-guardrails template.

Template path:
TEMPLATE_PATH_PLACEHOLDER

Follow this workflow:
1. Copy .cursor/commands/ from the template into this project (create the folder if it does not exist).
2. Capture a pre-upgrade baseline: run typecheck, lint, build, and test (skip any that do not exist). Save output to .cursor/guardrail-baseline.log.
3. Commit everything as: chore: snapshot before guardrail upgrade
4. Compare both projects and show a gap analysis table by layer.
5. Ask me which layers to apply before changing anything.
6. Never overwrite src/, package.json, or tsconfig.json — merge only.
7. After Layers 1-5: run a code compliance audit (Layer 6) and report Blocker / Should-fix / Nit findings. Fix blockers before finishing.
8. Show the Layer 7 governance checklist.

PowerShell bootstrap (Option B)

$template = "TEMPLATE_PATH_PLACEHOLDER"
$dest = ".cursor\commands"
New-Item -ItemType Directory -Force -Path $dest | Out-Null
Copy-Item "$template\.cursor\commands\*" $dest -Recurse -Force
Write-Host "Commands bootstrapped. Now type /guardrail-upgrade in Cursor chat."
